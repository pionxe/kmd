# KMD Reader Android 需求文档

> 项目阶段：课程项目第一阶段 PRD
> 文档状态：草案
> 最近更新：2026-05-07

## 1. 项目背景

KMD 是一种面向动态文本、舞台演出和交互式阅读体验的内容格式。当前 KMD 主要以 Web 编辑器形态存在，核心能力集中在语法解析、布局计算、特效管线和舞台渲染上。随着格式能力逐渐成型，KMD 需要一个更贴近普通读者的移动端入口，让创作内容不只停留在编辑器中，也可以作为作品被浏览、收藏和阅读。

KMD Reader Android 是 KMD 内容生态的移动端阅读宿主。它不在 Android 端重新实现 KMD 解析器，而是复用现有 Web Runtime 作为唯一解释核心，避免在 KMD 快速迭代阶段出现多套 parser、多套 layout 和多套渲染行为。Android 应用负责移动端产品体验，包括作品发现、书架管理、阅读进度、本地导入、离线缓存和阅读设置。

当前并不存在真实的 KMD 创作者社区，因此本项目第一版以“模拟社区内容源 + 本地示例作品”为基础实现完整阅读流程，并为未来接入真实社区 API 预留数据结构和页面边界。

## 2. 产品定位

KMD Reader Android 是一款面向 KMD 创作内容的移动阅读应用。用户可以在应用中浏览 KMD 作品列表，查看作品详情，打开作品进行阅读，并管理自己的收藏、阅读历史和本地导入内容。

产品关键词：

- KMD 内容阅读
- 移动端作品发现
- 书架与阅读进度
- 本地文件导入
- 面向未来社区的接口预留
- 单核心、多宿主架构

## 3. 产品目标

### 3.1 课程项目目标

- 完成一个具备完整产品闭环的 Android 阅读应用原型。
- 使用 Android 原生能力实现列表、详情、书架、设置、持久化和文件导入等功能。
- 通过 WebView 承载 KMD Web Runtime，完成 KMD 文件阅读能力。
- 提供清晰的 PRD、架构设计、数据模型和测试说明，支撑课程结项展示。

### 3.2 KMD 生态目标

- 验证 KMD 内容在移动端消费场景中的可行性。
- 为未来 KMD 社区、作品发布和移动端阅读体验建立产品骨架。
- 在不制造第二套核心实现的前提下，为 KMD Reader 提供跨平台宿主方案。

## 4. 目标用户

### 4.1 普通读者

对动态文本、视觉小说、互动诗歌、实验性阅读内容感兴趣，希望通过手机浏览和阅读 KMD 作品。

核心诉求：

- 快速找到感兴趣的作品。
- 阅读体验稳定、流畅、不会丢失进度。
- 可以收藏作品并继续阅读。

### 4.2 KMD 创作者

使用 KMD 编辑器创作内容，希望在移动端预览作品的实际阅读效果，或把作品展示给他人。

核心诉求：

- 导入本地 KMD 文件进行阅读预览。
- 阅读器表现与 Web 编辑器尽量一致。
- 能看到标题、作者、标签、简介等作品信息。

### 4.3 课程评审者

关注项目是否具备明确需求、合理技术方案、实际工作量和可演示成果。

核心诉求：

- 产品定位清晰。
- 功能范围可落地。
- Android 端工作量充分，不只是简单网页套壳。
- 技术风险有分析和应对策略。

## 5. 使用场景

### 5.1 浏览社区作品

用户打开应用后进入发现页，看到由模拟社区数据提供的推荐作品列表。用户可以按分类、标签或关键词筛选作品，点击作品进入详情页。

### 5.2 阅读 KMD 作品

用户在作品详情页点击“开始阅读”，应用打开阅读器页面。Android 侧将 KMD 文本、主题、字号和上次阅读进度传入 Web Runtime，Web Runtime 负责解析与渲染。阅读过程中，阅读器将进度回传给 Android 侧保存。

### 5.3 管理书架

用户可以收藏作品，收藏后的作品出现在书架页。书架页同时展示最近阅读记录，方便用户继续未完成作品。

### 5.4 本地导入 KMD 文件

创作者或测试用户可以从手机存储中选择 `.kmd` 文件导入应用。导入后，应用将文件作为本地作品保存，并允许用户从详情页或书架页打开阅读。

### 5.5 调整阅读偏好

用户可以在设置页调整阅读主题、字号、是否自动保存进度等偏好。设置项通过本地持久化保存，并在下次进入阅读器时自动生效。

## 6. 功能范围

### 6.1 MVP 必须实现

| 模块 | 功能 | 说明 |
|------|------|------|
| 发现页 | 作品列表 | 使用本地 mock 数据模拟社区作品 |
| 发现页 | 搜索 | 按标题、作者、标签搜索 |
| 发现页 | 分类/标签筛选 | 支持基础分类筛选 |
| 作品详情 | 基本信息展示 | 标题、作者、简介、封面、标签、字数或预计阅读时长 |
| 作品详情 | 开始/继续阅读 | 根据阅读历史决定按钮状态 |
| 书架 | 收藏作品 | 展示用户收藏的作品 |
| 书架 | 最近阅读 | 展示阅读过的作品与进度 |
| 阅读器 | KMD 渲染 | 通过 WebView 加载 KMD Web Runtime |
| 阅读器 | 阅读进度保存 | Web Runtime 回传进度，Android 侧持久化 |
| 阅读器 | 阅读设置应用 | 支持字号、主题等基础参数 |
| 本地导入 | 导入 `.kmd` 文件 | 将文件加入本地作品库 |
| 设置 | 阅读偏好 | 保存主题、字号等偏好 |
| 数据层 | 本地持久化 | 保存收藏、历史、导入作品和设置 |

### 6.2 可选增强

- 作品详情中的作者主页。
- 作品评分或点赞展示。
- 阅读器夜间模式。
- 最近导入文件列表。
- 错误日志页面，用于展示 KMD 解析或渲染失败信息。
- 网络 API mock 层，为未来真实社区接口做替换准备。

### 6.3 暂不实现

- 用户注册与登录。
- 真实作品上传与发布。
- 评论、关注、私信等社区互动。
- 推荐算法。
- 付费、打赏或会员体系。
- Kotlin 原生 KMD parser 或原生 KMD renderer。

## 8. 核心用户流程

### 8.1 社区作品阅读流程

```text
打开应用
  -> 进入发现页
  -> 浏览或搜索作品
  -> 进入作品详情
  -> 点击开始阅读
  -> 阅读器加载 KMD 内容
  -> 阅读过程中保存进度
  -> 返回书架后可继续阅读
```

### 8.2 本地作品导入流程

```text
进入本地导入
  -> 选择 .kmd 文件
  -> 应用读取文件内容
  -> 生成本地作品记录
  -> 进入作品详情
  -> 打开阅读器
```

### 8.3 收藏流程

```text
进入作品详情
  -> 点击收藏
  -> 写入本地收藏记录
  -> 作品出现在书架
  -> 再次点击可取消收藏
```

## 10. WebView 桥接需求

KMD Reader Android 采用单核心、多宿主架构。Android 端不拥有 KMD parser、layout 或 renderer 的第二份实现，而是通过 WebView 加载打包后的 KMD Reader Web Runtime。

### 10.1 Android 到 Web Runtime

Android 侧需要向 Web Runtime 传递：

- KMD 文本内容或内容 URL。
- 作品 ID。
- 阅读主题。
- 字号参数。
- 上次阅读进度。
- 可选调试开关。

### 10.2 Web Runtime 到 Android

Web Runtime 需要向 Android 侧回传：

- 当前阅读进度。
- 当前章节或位置标识。
- 作品标题等解析出的元信息。
- 解析失败或渲染失败错误。
- 阅读器加载完成事件。

### 10.3 错误处理

当 Web Runtime 解析或渲染失败时，Android 侧应展示错误状态，并提供返回、重试和查看错误摘要的入口。错误信息需要保存到日志中，便于课程演示和后续调试。

## 11. 非功能需求

### 11.1 性能

- 作品列表首屏加载应在 1 秒内完成。
- 阅读器页面应展示加载状态，避免空白等待。
- 本地 mock 作品和导入作品应支持离线访问。

### 11.2 一致性

- KMD 文件解释结果应以 Web Runtime 为准。
- Android 端不得维护独立 KMD parser，避免行为分叉。
- 若 Web 编辑器和 Android 阅读器表现不一致，应优先在 KMD Web Runtime 中定位问题。

### 11.3 可维护性

- Android 应用层、数据层、阅读器桥接层应分离。
- Mock 数据源应可替换为真实远程 API。
- Web Runtime 构建产物应有明确版本标识。

### 11.4 可用性

- 无作品、无收藏、导入失败、解析失败等状态应有明确提示。
- 阅读进度应自动保存，避免用户丢失阅读位置。
- 主要阅读流程不依赖网络。

## 12. 技术方案概述

推荐技术栈：

- Kotlin
- Jetpack Compose
- Navigation Compose
- ViewModel
- Room
- DataStore
- Android WebView
- 本地 JSON mock 数据

推荐目录形态：

```text
kmd-editor/
  docs/
    prd/
      kmd-reader-android-prd.md
  apps/
    android-reader/
      app/
      reader-runtime/
```

`reader-runtime/` 用于放置由 KMD Web 项目构建出的阅读器静态资源。课程项目初期可以使用构建产物复制方式，后续再演进为自动构建和包引用。

## 14. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| KMD 格式仍在快速变化 | Android 阅读器可能频繁适配 | 保持单一 Web Runtime 核心，Android 只消费构建产物 |
| WebView 被误认为工作量不足 | 影响课程评价 | Android 侧实现完整产品骨架、数据层、持久化和导入流程 |
| Reader Runtime 尚未独立 | 接入初期需要临时打包方案 | 先复制构建产物，后续抽出 `kmd-reader-web` |
| 移动端性能不稳定 | 阅读体验受影响 | 限制 MVP 样例复杂度，增加加载状态和错误提示 |
| 没有真实社区 | 作品发现缺少真实数据 | 使用本地 mock 数据模拟社区，保留 API 替换边界 |

## 16. 术语表

| 术语 | 说明 |
|------|------|
| KMD | 项目自定义的动态文本内容格式 |
| KMD Core | KMD 的解析、布局、效果和渲染核心能力 |
| Web Runtime | 在 Web 环境中运行的 KMD 阅读器内核 |
| 宿主 | 承载 KMD Runtime 的应用环境，例如 Web 编辑器或 Android Reader |
| Mock 社区 | 使用本地静态数据模拟的作品社区 |
| 单核心、多宿主 | 只维护一套 KMD 解释核心，由多个应用宿主复用 |
